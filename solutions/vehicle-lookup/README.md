# Vehicle Lookup

Gartner BOAT MQ 2026 demo solution. Demonstrates AI-assisted process development:

1. **Web Modeler Copilot** generates the process (Slack start → NHTSA lookup → risk assessment → XOR → Slack result)
2. **FEEL Copilot** writes the gateway condition
3. **Claude Code** fetches this BPMN, generates the job worker for the risk assessment service task, deploys, and runs a process instance
4. **Operate** shows the instance running
5. Commit + CI (bpmnlint, variable-schema) + merge to production

## Process variables

| Variable | Type | Source |
|---|---|---|
| `vin` | string | Slack webhook payload |
| `vehicleApiResponse` | object | NHTSA HTTP connector |
| `vehicleInfo` | object | Risk assessment worker |
| `riskScore` | number | Risk assessment worker |
| `eligible` | boolean | Risk assessment worker |

## Element templates

Templates are in `.element-templates/`. Import into Web Modeler or place in `~/.camunda/element-templates/` for Desktop Modeler.

- **NHTSA Vehicle Lookup**: wraps the HTTP JSON connector, pre-fills the NHTSA VIN decode endpoint
- **Vehicle Risk Assessment**: locks the job type `io.camunda.demo:vehicle-risk-assessment`; worker is generated by Claude Code during the demo

## Job worker

The worker for `io.camunda.demo:vehicle-risk-assessment` is generated by Claude Code during the demo. It:

1. Receives `vehicleApiResponse` (full NHTSA HTTP response)
2. Flattens `vehicleApiResponse.body.Results` (array of `{Variable, Value}` pairs) into a clean object
3. Computes `riskScore` based on vehicle type and GVWR
4. Sets `eligible = riskScore < 70`
5. Completes the job with `vehicleInfo`, `riskScore`, `eligible`
